The File Uploader can be configured using various options that control upload behavior, validation, UI appearance, and more. This page provides a complete reference for all available configuration options.

## Basic Configuration

<ParamField path="pubkey" type="string" required>
  Your project's Public Key. This is required to use the File Uploader.
  
  **Default:** `''`
</ParamField>

<ParamField path="multiple" type="boolean">
  Allow multiple file uploads.
  
  **Default:** `true`
  
  ```html
  <uc-config pubkey="YOUR_PUBLIC_KEY" multiple="true"></uc-config>
  ```
</ParamField>

<ParamField path="multipleMin" type="number">
  Minimum number of files to upload.
  
  **Default:** `0`
</ParamField>

<ParamField path="multipleMax" type="number">
  Maximum number of files to upload.
  
  **Default:** `Number.MAX_SAFE_INTEGER`
  
  ```html
  <uc-config pubkey="YOUR_PUBLIC_KEY" multiple-max="10"></uc-config>
  ```
</ParamField>

<ParamField path="confirmUpload" type="boolean">
  Require user confirmation before uploading.
  
  **Default:** `false`
</ParamField>

<ParamField path="groupOutput" type="boolean">
  Group output files. When enabled, multiple files will be combined into a single group.
  
  **Default:** `false`
</ParamField>

## File Validation

<ParamField path="imgOnly" type="boolean">
  Allow only image files.
  
  **Default:** `false`
  
  ```html
  <uc-config pubkey="YOUR_PUBLIC_KEY" img-only="true"></uc-config>
  ```
</ParamField>

<ParamField path="accept" type="string">
  Native file input accept attribute value. Also affects client validation settings.
  
  **Default:** `''`
  
  ```html
  <uc-config accept="image/*,.pdf"></uc-config>
  ```
</ParamField>

<ParamField path="maxLocalFileSizeBytes" type="number">
  Maximum size of local files in bytes. Set to `0` for no limit.
  
  **Default:** `0`
  
  ```html
  <uc-config max-local-file-size-bytes="10485760"></uc-config>
  ```
</ParamField>

<ParamField path="fileValidators" type="FileValidators">
  Validators for individual files. This is an array of file validator functions.
  
  **Default:** `[]`
  
  <Note>
  This is a complex type that must be set via JavaScript, not as an HTML attribute.
  </Note>
  
  ```javascript
  const config = document.querySelector('uc-config');
  config.fileValidators = [
    (file) => {
      if (file.size > 1000000) {
        return { message: 'File is too large' };
      }
    }
  ];
  ```
</ParamField>

<ParamField path="collectionValidators" type="CollectionValidators">
  Validators for file collections. This is an array of collection validator functions.
  
  **Default:** `[]`
  
  <Note>
  This is a complex type that must be set via JavaScript, not as an HTML attribute.
  </Note>
</ParamField>

<ParamField path="validationTimeout" type="number">
  Timeout for async validation functions, in milliseconds.
  
  **Default:** `15000` (15 seconds)
</ParamField>

<ParamField path="validationConcurrency" type="number">
  The number of files to validate concurrently.
  
  **Default:** `100`
</ParamField>

## Upload Sources

<ParamField path="sourceList" type="string">
  List of sources for file uploads. Comma-separated list of source names.
  
  **Default:** `'local, url, camera, dropbox, gdrive'`
  
  **Available sources:**
  - `local` - Local file system
  - `url` - URL import
  - `camera` - Camera capture
  - `dropbox` - Dropbox
  - `gdrive` - Google Drive
  - `gphotos` - Google Photos
  - `instagram` - Instagram
  - `facebook` - Facebook
  - `flickr` - Flickr
  - `vk` - VK
  - `evernote` - Evernote
  - `box` - Box
  - `onedrive` - OneDrive
  - `huddle` - Huddle
  
  ```html
  <uc-config source-list="local, url, camera"></uc-config>
  ```
</ParamField>

<ParamField path="sourceListWrap" type="boolean">
  Wrap the source list when it overflows.
  
  **Default:** `true`
</ParamField>

<ParamField path="externalSourcesPreferredTypes" type="string">
  Preferred types for external sources. This affects which file types are displayed first in external source pickers.
  
  **Default:** `''`
  
  See [External Sources Preferred Types](https://uploadcare.com/docs/file-uploader/options/#external-sources-preferred-types) for more details.
</ParamField>

<ParamField path="externalSourcesEmbedCss" type="string">
  Provide custom CSS to the social sources iframe.
  
  **Default:** `''`
</ParamField>

<ParamField path="remoteTabSessionKey" type="string">
  Key to revoke Custom OAuth access. See [OAuth docs](https://uploadcare.com/docs/upload-sources/#oauth) for details.
  
  **Default:** `''`
</ParamField>

<ParamField path="topLevelOrigin" type="string">
  Top-level origin for the uploader. This is used for Google Drive Picker if there is no access to the origin due to the cross-origin policy.
  
  **Default:** `''`
</ParamField>

## Camera Configuration

<ParamField path="cameraCapture" type="'user' | 'environment' | ''">
  Default camera capture mode. Determines which camera is used by default.
  
  **Default:** `''`
  
  - `'user'` - Front-facing camera
  - `'environment'` - Back-facing camera
  - `''` - Browser default
</ParamField>

<ParamField path="cameraMirror" type="boolean">
  Mirror the camera view horizontally.
  
  **Default:** `false`
</ParamField>

<ParamField path="cameraModes" type="string">
  The camera modes to enable in the camera modal. It is possible to select photo or video capture. The first mode is the default mode.
  
  **Default:** `'photo, video'`
  
  ```html
  <uc-config camera-modes="photo"></uc-config>
  ```
</ParamField>

<ParamField path="defaultCameraMode" type="'photo' | 'video' | null">
  The default tab to open in the camera modal.
  
  **Default:** `null`
  
  <Warning>
  **Deprecated:** Use `cameraModes` instead.
  </Warning>
</ParamField>

<ParamField path="enableAudioRecording" type="boolean">
  Enable audio recording in the camera modal.
  
  **Default:** `true`
</ParamField>

<ParamField path="enableVideoRecording" type="boolean | null">
  Enable video recording.
  
  **Default:** `null`
  
  <Warning>
  **Deprecated:** Use `cameraModes` instead.
  </Warning>
</ParamField>

<ParamField path="maxVideoRecordingDuration" type="number | null">
  The maximum duration of the video recording in seconds. Set to `null` for no limit.
  
  **Default:** `null`
</ParamField>

<ParamField path="mediaRecorderOptions" type="MediaRecorderOptions | null">
  A dictionary object that can contain properties from MediaRecorderOptions.
  
  **Default:** `null`
  
  <Note>
  This is a complex type that must be set via JavaScript, not as an HTML attribute.
  </Note>
  
  ```javascript
  const config = document.querySelector('uc-config');
  config.mediaRecorderOptions = {
    mimeType: 'video/webm;codecs=vp9',
    videoBitsPerSecond: 2500000
  };
  ```
</ParamField>

<ParamField path="cdnCnamePrefixed" type="string">

  **Default:** `'https://ucarecd.net'`
</ParamField>

  **Default:** `26214400` (25 MB)

  **Default:** `4`
</ParamField>

<ParamField path="multipartMaxAttempts" type="number">

  **Default:** `3`
</ParamField>

## Security

<ParamField path="secureSignature" type="string">
  Secure signature for uploads. Used for signed uploads to prevent unauthorized file uploads.
  
  **Default:** `''`
  
  See [Signed Uploads](https://uploadcare.com/docs/security/secure-uploads/) for more details.
</ParamField>

<ParamField path="secureExpire" type="string">
  Expiry time for secure uploads. Unix timestamp as a string.
  
  **Default:** `''`
</ParamField>

<ParamField path="secureUploadsExpireThreshold" type="number">
  Expiry threshold for secure uploads in milliseconds. When the signature is about to expire within this threshold, the resolver will be called again.
  
  **Default:** `600000` (10 minutes)
</ParamField>

<ParamField path="secureUploadsSignatureResolver" type="SecureUploadsSignatureResolver | null">
  Resolver function for secure uploads signature. Returns a promise that resolves to an object containing `secureSignature` and `secureExpire`.
  
  **Default:** `null`
  
  <Note>
  This is a complex type that must be set via JavaScript, not as an HTML attribute.
  </Note>
  
  ```javascript
  const config = document.querySelector('uc-config');
  config.secureUploadsSignatureResolver = async () => {
    const response = await fetch('/get-uploadcare-signature');
    const { signature, expire } = await response.json();
    return {
      secureSignature: signature,
      secureExpire: expire
    };
  };
  ```
</ParamField>

<ParamField path="secureDeliveryProxy" type="string">
  Proxy URL for secure delivery.
  
  **Default:** `''`
</ParamField>

<ParamField path="secureDeliveryProxyUrlResolver" type="SecureDeliveryProxyUrlResolver | null">
  Resolver function for secure delivery proxy URL.
  
  **Default:** `null`
  
  <Note>
  This is a complex type that must be set via JavaScript, not as an HTML attribute.
  </Note>
  
  ```javascript
  const config = document.querySelector('uc-config');
  config.secureDeliveryProxyUrlResolver = async (previewUrl, { uuid, cdnUrlModifiers, fileName }) => {
    return `https://your-proxy.com/${uuid}/${cdnUrlModifiers}/${fileName}`;
  };
  ```
</ParamField>

## Metadata

<ParamField path="metadata" type="Metadata | MetadataCallback | null">
  Metadata for uploaded files. Can be a static object or a callback function that returns metadata for each file.
  
  **Default:** `null`
  
  <Note>

  ```javascript
  // Static metadata for all files
  const config = document.querySelector('uc-config');
  config.metadata = {
    userId: '12345',
    source: 'web-app'

  config.metadata = async (fileEntry) => {
    return {
      fileName: fileEntry.name,
      uploadedAt: new Date().toISOString()

  };
  ```

## Configuration Methods

### Setting Options via HTML Attributes

Simple configuration options can be set directly as HTML attributes. Use kebab-case for attribute names:

```html
<uc-config
  pubkey="YOUR_PUBLIC_KEY"
  multiple="true"
  multiple-max="10"
  img-only="false"
  max-local-file-size-bytes="10485760"
></uc-config>
```

### Setting Options via JavaScript

Complex options (objects, functions, arrays) must be set via JavaScript:

```javascript
const config = document.querySelector('uc-config');

// Set simple options
config.multiple = true;
config.multipleMax = 10;

// Set complex options
config.metadata = { userId: '12345' };
config.fileValidators = [
  (file) => {
    if (file.size > 1000000) {
      return { message: 'File is too large' };
    }
  }
];
```

## Type Definitions

### FileValidator

```typescript
type FuncFileValidator = (
  fileEntry: OutputFileEntry
) => OutputError<OutputFileErrorType> | void | Promise<OutputError<OutputFileErrorType> | void>;

type FileValidatorDescriptor = {
  type: OutputFileErrorType;
  validator: FuncFileValidator;
};

type FileValidator = FuncFileValidator | FileValidatorDescriptor;
type FileValidators = FileValidator[];
```

### CollectionValidator

```typescript
type FuncCollectionValidator = (
  collection: OutputFileEntry[]
) => OutputError<OutputCollectionErrorType> | void | Promise<OutputError<OutputCollectionErrorType> | void>;

type CollectionValidators = FuncCollectionValidator[];
```

### MetadataCallback

```typescript
type Metadata = Record<string, string>;

type MetadataCallback = (
  fileEntry: OutputFileEntry
) => Promise<Metadata> | Metadata;
```

### SecureUploadsSignatureResolver

```typescript
type SecureUploadsSignatureAndExpire = {
  secureSignature: string;
  secureExpire: string;
};

type SecureUploadsSignatureResolver = () => Promise<SecureUploadsSignatureAndExpire | null>;
```

### SecureDeliveryProxyUrlResolver

```typescript
type SecureDeliveryProxyUrlResolver = (
  previewUrl: string,
  urlParts: {
    uuid: string;
    cdnUrlModifiers: string;
    fileName: string;
  }
) => Promise<string> | string;
```

### IconHrefResolver

```typescript
type IconHrefResolver = (iconName: string) => string;
```

### LocaleDefinitionOverride

```typescript
type LocaleDefinitionOverride = Record<string, Partial<LocaleDefinition>>;
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Auto Store for Production">
    Set `store` to `'auto'` to use your project's default storage setting. This makes it easier to manage storage behavior across different environments.
    
    ```html
    <uc-config store="auto"></uc-config>
    ```
  </Accordion>
  
  <Accordion title="Validate Files Early">
    Use `fileValidators` to validate files before they're uploaded. This provides immediate feedback to users and reduces unnecessary upload traffic.
    
    ```javascript
    config.fileValidators = [
      (file) => {
        // Check file size
        if (file.size > 5 * 1024 * 1024) {
          return { message: 'File must be smaller than 5MB' };
        }
        
        // Check file type
        if (!file.mimeType.startsWith('image/')) {
          return { message: 'Only images are allowed' };
        }
      }
    ];
    ```
  </Accordion>
  
  <Accordion title="Use Secure Uploads for Sensitive Applications">
    Implement secure uploads using `secureUploadsSignatureResolver` to prevent unauthorized file uploads.
    
    ```javascript
    config.secureUploadsSignatureResolver = async () => {
      const response = await fetch('/api/uploadcare-signature');
      return await response.json();
    };
    ```
  </Accordion>
  
  <Accordion title="Optimize Multipart Upload Settings">
    Adjust multipart upload settings based on your users' typical connection speeds. For users with slower connections, consider using smaller chunk sizes.
    
    ```html
    <uc-config
      multipart-min-file-size="10485760"
      multipart-chunk-size="2097152"
    ></uc-config>
    ```
  </Accordion>
  
  <Accordion title="Customize Source List">
    Only include the upload sources that your users actually need. This simplifies the UI and improves the user experience.
    
    ```html
    <uc-config source-list="local, camera, url"></uc-config>
    ```
  </Accordion>
  
  <Accordion title="Add Metadata for Organization">
    Use the `metadata` option to attach custom data to uploaded files. This helps with organizing and tracking files in your application.
    
    ```javascript
    config.metadata = {
      userId: currentUser.id,
      context: 'profile-photo',
      uploadedFrom: 'web-app'
    };
    ```
  </Accordion>
</AccordionGroup>
